## Design Context

### Users

Solo developer (the creator) who does a monthly end-of-month / start-of-month financial review at a laptop. Not a daily driver — a deliberate monthly ritual. Technically fluent, comfortable with density and data. Expects precision over hand-holding.

### Brand Personality

**Three words**: precise, composed, personal.

This is a financial operating system for one person. It should feel like a well-designed accountant's ledger that someone actually cares about — not a generic fintech dashboard, not a consumer budgeting app, not a startup SaaS product. It has the calm confidence of a tool built for exactly one job, by someone who knows what they want.

### Aesthetic Direction

- **Theme**: Light mode. Off-white/cream background (not pure white — tinted slightly toward the brand hue), dark ink text.
- **Color**: Largely monochromatic with one strong accent: deep cobalt/navy blue (oklch ~0.38 0.16 265). Used sparingly — active states, CTAs, links, accent labels. Green for income/credits (oklch ~0.55 0.15 142), red for expenses/debits (destructive, already in theme). No gradient text, no glow, no glassmorphism.
- **Layout**: Spreadsheet-inspired — tabular, left-aligned, precise grid columns. Dense but breathable. Data rows should feel like rows in a well-designed spreadsheet, not "cards."
- **Typography**: Clean, geometric sans with excellent tabular numeral support. Geist Variable (already installed) works well for UI and data. Page titles can use a heavier weight for hierarchy. Numbers must always use `tabular-nums` feature.
- **References**:
  - Cloudflare Agents site — controlled whitespace, developer-forward, confident simplicity
  - Loot app — cobalt accent on off-white, transaction rows with clear hierarchy, inline categorization, clean sidebar nav, monospace-adjacent data presentation
- **Anti-references**: Consumer budgeting apps (Mint, YNAB's bubbly aesthetic), crypto dashboard aesthetics (dark mode neon), generic admin panels.

### Design Principles

1. **Data first, chrome last** — Every pixel of UI that isn't data is overhead. Minimize decoration; maximize information density.
2. **Spreadsheet clarity** — Column alignment is sacred. Numbers right-align. Labels left-align. Amounts always show currency and sign. No ambiguity.
3. **One accent, used with restraint** — The cobalt blue earns its place by being rare. Interactive elements get it; decorative elements don't.
4. **Light and grounded** — Off-white background with ink-dark text. Feels like paper, not a screen. Green/red for income/expense signal, not decoration.
5. **Built for revisiting** — Monthly review context means layouts should orient fast. The user already knows the app — optimize for scan speed over onboarding.
